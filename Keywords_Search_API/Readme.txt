1. 确定是否已打开 ElasticSearch 本地服务器
2. 保证运行环境可用内存超过7G或8G，因为wf_article表中数据超过2000W，一次性读取出来会占用较多内存
3. 直接运行 index_building.py 文件即可，该文件当前大概需要执行30分钟左右
4. 最好熟悉一下 index_building.py 文件里的内容，以后如果需要再添加其他表或字段的索引，还需要手动增加函数调用代码
5. index_building.py 文件中 build 函数的第一个参数 post_batch 需要人工决定，该参数大小不同时，每一批数据在当前文件夹下生成的临时数据文件 json_data_temp.json 的文件大小不同，ES的批量导入规定该文件大小不能超过 100M，否则就会导入失败(但是不会报错)，所以在决定该参数大小的时候，一定要保证每批数据生成的临时数据文件大小不超过100M！这点非常重要！可以通过 curl -X GET 'http://localhost:9200/_cat/count/index_name?v' 命令来检查导入之后的索引条数，看是否全部导入成功。
